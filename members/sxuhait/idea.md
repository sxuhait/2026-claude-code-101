# 💡 프로젝트 기획서 — <프로젝트 이름>

| 항목 | 내용 |
| --- | --- |
| **1. 한 줄 정의** | 지역아동센터 및 교육 지원 멘토들을 위한 '멘토링 일지 및 멘티 성장 관리' 자동화 웹서비스 |
| **2. 문제** | 멘토링 직후 매번 수기로 일지를 쓰고 기관에 제출하는 과정이 번거로움. 또한, 여러 멘티들의 이전 학습 진도나 상담 내용이 한곳에 누적되지 않아 연속성 있는 멘토링이 어려움. |
| **3. 해결책** | 스마트폰으로 간편하게 일지를 입력하면, 누적된 데이터를 통해 멘티별 성장 히스토리를 타임라인으로 보여주고 월말 제출용 일지를 자동 취합해 줌 |
| **4. 왜 지금** | AI(Claude API 등)를 활용해 입력된 일지 텍스트를 요약하거나 기관 제출용 포맷으로 자동 변환하는 기능을, 6주 안에 개인 개발자도 충분히 구현할 수 있는 환경이 됨. |
| **5. 사용자** | 지역아동센터, 학교 등에서 장학생이나 봉사활동으로 청소년 멘토링을 진행하는 전국의 대학생 멘토들. |
| **6. 대안·경쟁** | 엑셀, 한글 파일, 수기 장부. 차별점은 모바일에서 이동 중에도 즉시 작성 가능한 편의성과 '멘티별 히스토리'를 한눈에 모아보는 특화된 UX. |
| **7. 제품** | 핵심 기능: ① 모바일 친화적 일지 간편 작성 폼 ② 멘티별/날짜별 히스토리 조회 ③ 월말 제출용 일지 내보내기. (6주 MVP 범위: ①, ② 우선 구현) |
| **8. 지속 방법** | 대학생 멘토 개인에게는 무료로 배포하여 바이럴로 사용자를 확보. 추후 체계적인 멘토 관리가 필요한 지역아동센터 등 복지/교육 기관에 B2B 대시보드로 확장. |
| **9. 나(팀)** | 현재 지역아동센터에서 청소년 멘토링을 진행 중이며, 졸업한 모교 진로 멘토링 등 현장 경험이 풍부해 멘토들이 실제로 겪는 '귀찮음'과 '필요한 기능'을 누구보다 정확히 앎. |
| **10. 6주 계획** | 3주: DB 스키마 설계 및 간편 일지 입력 폼 구현 / 4주: 멘티별 히스토리 타임라인 로직 개발 / 5주: 월말 내보내기 기능 및 UI 다듬기 / 6주: 서비스 배포 및 피드백 수집 |
