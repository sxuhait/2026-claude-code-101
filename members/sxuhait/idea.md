# 💡 프로젝트 기획서 — <새싹일기>

| 항목 | 내용 |
| --- | --- |
| **1. 한 줄 정의** | 지역아동센터 멘토를 위한, 누적된 일지 데이터를 분석해 아동별 학습 취약점과 다음 지도 방향을 먼저 제안해주는 AI 멘토링 코파일럿 |
| **2. 문제** | 멘토들은 일지를 써도(혹은 안 써도) 여러 회차에 걸친 기록을 다시 들여다보며 "이 아이가 최근 뭘 어려워하는지, 뭘 더 챙겨야 하는지" 패턴을 스스로 찾아낼 시간과 여유가 없음. 그 결과 아이의 학습 취약점 파악과 지도가 멘토 개인의 기억력·감에 의존하게 되고, 문제가 누적돼도 늦게 발견됨. 여기에 더해 아이가 정해진 요일에 오는지, 결석이 반복되는지도 별도로 관리가 안 돼서 놓치기 쉬움. (다른 지역아동센터 멘토 인터뷰에서도 동일하게 "출석/스케줄 관리 필요"가 언급됨 — 독립적으로 나온 신호라 문제의 신뢰도를 뒷받침함). |
| **3. 해결책** | 모바일로 1분 만에 폼을 채워 작성 허들을 낮추고, 누적된 일지·출석 데이터를 AI가 분석해 아동별 학습 취약점과 다음 지도 포인트를 먼저 제안함. |
| **4. 왜 지금** | LLM이 비정형 텍스트(일지)에서 패턴을 추출하고 실용적 수준의 제안을 생성하는 정확도가 최근에야 개인 개발자도 API로 접근 가능한 수준이 됨. 여기에 청소년 멘토링 프로그램 참여가 늘면서 멘토 1인당 관리 부담도 함께 늘고 있음. |
| **5. 사용자** | 온새미로지역아동센터/대학생청소년멘토링, 다문화멘토링 소속 대학생 멘토 8명 — 여러 아이를 동시에 담당하며 회차별 변화를 스스로 추적하기 어려운 멘토 |
| **6. 대안·경쟁** | 종이 일지, 수기 장부, 기관 자체 엑셀 양식. 이들은 모두 "기록"만 남길 뿐 "취약점 파악"은 여전히 멘토의 몫으로 남겨둠. 차별점은 기록 이후 AI가 패턴을 읽어 먼저 알려준다는 것 — 경쟁은 로깅 툴이 아니라 "판단을 대신 도와주는 도구"가 없다는 공백. |
| **7. 제품** | 핵심 기능: ① 아동별 요일 등록 + 출석 체크(가벼운 버전 — 캘린더/일정변경 없이 "이 요일에 오기로 함" 등록과 체크만) ② 템플릿 기반 모바일 간편 일지(진도/특이사항) 작성 ③ AI가 누적 일지+출석 데이터를 분석해 아동별 학습 취약점과 결석 패턴을 진단하고 다음 회차 지도 포인트를 제안하는 대시보드 ④ 월말 제출용 일지 자동 요약·포맷 변환. |
| **8. 지속 방법** | 대학생 멘토 개인에게는 무료로 배포하여 바이럴로 사용자를 확보. 추후 체계적인 멘토 관리가 필요한 지역아동센터 등 복지/교육 기관에 B2B 대시보드로 확장. |
| **9. 나(팀)** | 현재 지역아동센터에서 직접 멘토링을 진행하며 '기록은 남기지만 패턴은 못 읽는' 한계를 현장에서 체감하고 있어, 멘토에게 실제로 필요한 진단·제안 기능을 우선순위 있게 구현할 수 있음. |
| **10. 6주 계획** | 1~2주: (완료) 환경 설치 및 Claude Code 기초 실습 / 3주: DB 스키마 설계(멘토·멘티·일지·출석·회차별 이력) + 로그인/인증 + 요일 등록·출석 체크 + 간편 일지 입력 폼 구현 / 4주: AI 취약점 진단·제안 엔진 프로토타입 개발 (일지+출석 누적 데이터 → 요약/패턴/결석 알림/제안 생성) / 5주: 멘티별 히스토리 타임라인 + 월말 내보내기 + UI 다듬기 / 6주: 서비스 배포 및 실제 멘토 피드백 수집. 성공 기준: 멘토 5명이 2주 이상 사용 + AI 제안에 대해 "도움됐다"는 응답 70% 이상 |
